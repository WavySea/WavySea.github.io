切图

  • 修饰性的：background属性--图标、logo、有特殊效果的按钮、文字、非纯色背景等。（一般存为png格式）

3.在X轴方向上平铺时，需要充满新建画布的宽。

保存和维护

保存

  • 若需要的内容是已经被合并在一起的情况：矩形——魔棒-----Ctrl + C —-Ctrl + N ——Ctrl +V 复制，新建，粘贴 到新的文件
  • 若需要的内容在独立图层，新建文件后，直接把该图拖入新文件
  • 存储为web所用格式——窗口 Alt + Shift + Ctrl +S (快捷键可以提高效率)
  • 保存类型：
  • JPG： 适合色彩丰富且无透明要求的图片；品质一般设为60-80之间，目的是一方面保留一定图片品质，同时缩小图片大小以便于网络传输；
  • PNG8：适合色彩不太丰富的图片，有或无透明要求均可；图片大小比较小，适合网络传输；在保存时注意杂边选为无，选无仿色，选好后可以保存设置留着以后用；
  • PNG24：适合半透明格式，因为PNG24不会压缩图片，因此一方面图片会保留原品质和色彩，另一方面大小会相对较大。
  • ps. 保留一份PSD： 便于后期维护，保留原图质量；

修改、维护

  1. 更改画布大小：图像—-画布大小 （PS.定位选在左上角）
  2. 移动图标：若为独立图层，用移动工具拖动；若为非独立图层，矩形选框选中，再用移动工具拖动；若想把两个图标分开为两个图层，选中后crlt + X剪切，再ctrl+v粘贴，就已经是分开图层了
  3. 减小画布：选取工具选中需要的区域，再裁剪（图像—裁剪）； 直接用裁剪工具
  4. 维护PNG8格式时，更改从索引颜色格式到RGB，以防丢失颜色信息：图像—模式—RGB；

图片优化与合并

使用背景图：
HTML： <button class=“u-btn”>click me </button>
CSS: .u-btn{background:url(images/btn.png) no-repeat 0 0;} .u-btn{background:url(images/sprite.png) no-repeat 0 -50;}(用于图标合并在一张图里的情况，相当于图片向上移动了50px)

图片合并方案

  • Sprite拼图： 优点：减少网络请求，提升网页加载速度；
  • 大小与质量：二者取平衡；
  • 压缩工具：无损 Minimage（图片大小无明显影响）有损Tinypng(图片大小有明显改变)
  • 合并排列：图片之间必须保留空隙（可维护性以便后续的改变，并且给css代码保留一定误差空隙即容错性） （对于具体要保留多大的空隙呢，并没有一个明确的数值，遵循的一个宗旨就是保留的空隙足够后期维护就可以了。如果是小图标，留的空隙可适当小一些，一般20像素左右；那如果是大图标，要留的空隙就要大一点，因为大图标在调整的时候，影响到的空间也会比较大。）
  • 排列方式：横向排列，纵向排列
  • 合并：
    (1) 同属一个模块的图片进行合并；
    (2) 大小相近的图片合并（利于利用空间）；
    (3) 色彩相近的图片合并（使图片大小较小）
    (4) 推荐：只本页用到的图片合并（后面的页面不用再加载这个图片）；
    (5) 有状态的图标合并（两个状态的图标差别只是x距离而已，便于后续操作）
    IE6不支持PNG24半透明或者CSS新性能，则存两份图片
